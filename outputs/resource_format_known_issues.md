# Resource Format Known Issues

## Статус

Этот файл фиксирует текущее практическое состояние общего resource-layer в PoC-проекте `Aurora KMP Demo`.

## Наблюдение 1. Android и SVG

В рамках текущей конфигурации проекта `SVG` пока нельзя считать беспроблемным общим drawable-форматом для `Android`.

Рабочая формулировка для проекта сейчас такая:

- Android не воспринимает `SVG` достаточно надёжно как shared drawable-format в этом PoC;
- поэтому `SVG` не считается закрытым и безопасным решением для всех target.

## Наблюдение 2. Aurora и XML drawable

В рамках текущей конфигурации проекта `Aurora` пока нельзя считать совместимой с shared `XML VectorDrawable` через используемый Compose Resources flow.

Рабочая формулировка для проекта сейчас такая:

- Aurora не воспринимает `XML drawable` достаточно надёжно в этом PoC;
- отдельные эксперименты с XML были вынесены в диагностический экран;
- при загрузке таких drawable на Aurora наблюдалось падение вида:
  - `kotlin.RuntimeException: Can't wrap nullptr`

## Важная оговорка

Это не окончательное утверждение в стиле:

- "Android вообще не поддерживает SVG";
- "Aurora вообще не поддерживает XML VectorDrawable".

Это более аккуратный и честный вывод:

- в текущем проекте;
- с текущими библиотеками;
- с текущей Compose Resources интеграцией;
- в текущем diagnostic flow

оба направления остаются проблемными и не считаются полностью закрытыми.

## Практический вывод для команды

Пока в проекте стоит исходить из следующего:

- resource-layer уже реально общий;
- строки и шрифты уже являются рабочими shared resources;
- drawable-слой остаётся зоной риска и требует дополнительных платформенных проверок;
- все изменения вокруг drawable-форматов нужно проверять отдельно на Android и Aurora, а не считать автоматически переносимыми.
