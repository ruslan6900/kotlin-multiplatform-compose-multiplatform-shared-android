# Release 0.0.2

Дата фиксации: `2026-07-03`

## Что вошло в 0.0.2

- общий слой ресурсов через `Compose Resources`
  - строки;
  - шрифты;
  - общие drawable-ресурсы;
- переходы между экранами через `Compose Navigation`;
- сохранён Aurora-friendly diagnostic bootstrap:
  - `Super Smoke`;
  - `Render Smoke`;
  - `Normal Demo`;
  - отдельный экран `Drawable Experiment Lab`;
- Room используется как текущий общий persistence backend;
- проведена серия повторных проверок поведения Aurora при запуске shared drawable-ресурсов.

## Что проверялось отдельно

В этой версии был сделан отдельный экспериментальный маршрут для диагностики общих drawable-ресурсов:

- `svg_painter`
- `xml_min_painter`
- `xml_two_paths_painter`
- `xml_stroke_painter`
- `xml_original_painter`
- `xml_min_vector`
- `xml_two_paths_vector`
- `xml_stroke_vector`
- `xml_original_vector`

Целью было понять:

- является ли проблема Aurora-specific только для сложного XML;
- поддерживает ли Aurora Compose Resources хотя бы минимальный XML VectorDrawable;
- отличается ли поведение `painterResource(...)` и `vectorResource(...)`.

## Текущее зафиксированное состояние по ресурсам

На текущем срезе проекта принимается следующий practical status:

- `Android` сейчас проблемно воспринимает `SVG` как основной shared drawable-формат в этом PoC;
- `Aurora` сейчас проблемно воспринимает `XML drawable` в диагностическом сценарии;
- это ещё не окончательный теоретический вывод о платформе вообще, но это актуальное состояние именно данного проекта и его текущего resource pipeline.

Иными словами, пока в проекте зафиксировано рабочее практическое наблюдение:

- `SVG` нельзя считать беспроблемным общим форматом для Android в текущей конфигурации проекта;
- `XML VectorDrawable` нельзя считать беспроблемным общим форматом для Aurora в текущей конфигурации проекта.

## Aurora XML investigation status

Для Aurora было выполнено неоднократное тестирование, чтобы разрешить проблему отсутствия стабильной поддержки `XML VectorDrawable`, но на текущий момент безуспешно.

Что удалось установить:

- проблема воспроизводится в отдельном drawable diagnostics flow;
- падение проявляется как `kotlin.RuntimeException: Can't wrap nullptr`;
- проблема была достаточно серьёзной, чтобы вынести её в отдельный экран диагностики, а не скрывать внутри основного UI.

## Зачем фиксировать это именно сейчас

Версия `0.0.2` нужна как стабильная контрольная точка, в которой уже подтверждены:

- общие ресурсы;
- общий navigation graph;
- общий shared UI flow;
- общий Room-based storage;
- наличие отдельной диагностической ветки для resource/runtime проблем Aurora.

При этом ограничение по `SVG/XML drawable` не замалчивается, а явно зафиксировано как известная проблема.
